[v0.4][fix:tts] Fix Cartesia API parameters, align with working scripts

PROBLEM:
- Worker crashed: "AsyncTTSResource.bytes() got an unexpected keyword argument 'experimental_controls'"
- TTS generation failed despite valid API keys
- Backend tidak selaras dengan working implementation di scripts/voice/

ROOT CAUSE:
- Backend used incorrect API parameter: experimental_controls inside voice dict
- Working scripts (scripts/voice/tts_lib.py) used correct format: generation_config
- Cartesia API rejected experimental_controls as unknown parameter

SOLUTION:
1. Removed experimental_controls from voice dict
2. Added speed parameter (top-level)
3. Added generation_config parameter with speed, volume, emotion
4. Aligned backend with working scripts format
