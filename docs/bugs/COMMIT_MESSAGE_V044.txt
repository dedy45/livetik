[v0.4][fix:env] Consolidate .env to root, fix TTS init error

PROBLEM:
- User reported "✗ TTS not initialized — check CARTESIA_API_KEYS"
- Valid Cartesia API key existed in livetik/.env
- Worker couldn't see the key

ROOT CAUSE:
- Duplicate .env files: livetik/.env (complete) + apps/worker/.env (partial)
- Worker used load_dotenv() without path → reads from CWD
- When run from apps/worker/, loaded apps/worker/.env (no API keys)
- CartesiaPool.from_env() raised RuntimeError: "CARTESIA_API_KEYS is empty"

SOLUTION:
1. Delete redundant apps/worker/.env (partial config)
2. Fix main.py to load .env from repo root explicitly
3. Use Path(__file__).parent traversal for CWD-independent config loading

- Modified: apps/worker/src/banghack/main.py
