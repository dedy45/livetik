[v0.4][fix:tts] Use direct HTTP API instead of SDK client, align with working scripts

PROBLEM:
- TypeError: 'async for' requires an object with __aiter__ method, got coroutine
- TTS generation failed on every attempt
- v0.4.5 fix (generation_config) wasn't enough

ROOT CAUSE (Systematic Debugging - Phase 1):
- client.tts.bytes() returns COROUTINE (needs await), not async iterator
- Backend used "async for" on coroutine → TypeError
- Working scripts (scripts/voice/tts_lib.py) use requests.post() → direct HTTP
- SDK client API unclear/changed in v2.0.0+

PATTERN ANALYSIS (Phase 2):
- Working scripts: requests.post() → response.content ✅
- Broken backend: AsyncCartesia.tts.bytes() → async for ❌
- Solution: Use direct HTTP API like working scripts

HYPOTHESIS (Phase 3):
"Backend should use direct HTTP API (httpx) instead of SDK client"
- Proven by working scripts
- Simpler, fewer dependencies
- Stable, documented API

SOLUTION (Phase 4):
1. Removed "from cartesia import AsyncCartesia"
2. Added "import httpx" for async HTTP calls
3. Replaced SDK client with direct HTTP API
4. Used same payload/headers as working scripts
5. Aligned backend with proven working code
